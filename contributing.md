# Contribution Guidelines

Thanks for your interest in making this list better! Please keep these guidelines in mind.

## Adding to the list

- Make an individual pull request for each suggestion.
- Add your entry to the most appropriate section, keeping entries alphabetical within a section where the section is already ordered that way.
- Use the format: `` - [name](link) - Description. `` — a title-cased name, then a short, objective description that ends with a period.
- The description should say what the thing *is*, not sell it. No marketing taglines.
- Only suggest resources that are genuinely useful, actively relevant, and real (a maintained repo, a published paper with a DOI, an official grammar, etc.).
- New entries should be things you have actually used or verified — not a link you found and assumed was good.
- Check your spelling and grammar.
- The pull request title should describe the change (e.g. `Add Foo runtime`), not `Update README`.

## Removing or fixing an entry

If a resource is dead, unmaintained, moved, or inaccurately described, a PR to fix or remove it is very welcome.

## Quality bar

This list aims to be a signal, not a dump. If in doubt about whether something belongs, open an issue to discuss it first.
